DataSourceSpecConfiguration describes the source data that an instrument wants to get from the
sourcing engine.

```graphql
type DataSourceSpecConfiguration {
  signers: [Signer!]
  filters: [Filter!]
}
```


### Fields

#### [<code style={{ fontWeight: 'normal' }}>DataSourceSpecConfiguration.<b>signers</b></code>](#)<Bullet />[`[Signer!]`](/testnet/api/graphql/objects/signer) <Badge class="secondary" text="list"/> <Badge class="secondary" text="object"/>
> signers is the list of authorized signatures that signed the data for this
data source. All the public keys in the data should be contained in this
list.
> 

#### [<code style={{ fontWeight: 'normal' }}>DataSourceSpecConfiguration.<b>filters</b></code>](#)<Bullet />[`[Filter!]`](/testnet/api/graphql/objects/filter) <Badge class="secondary" text="list"/> <Badge class="secondary" text="object"/>
> filters describes which source data are considered of interest or not for
the product (or the risk model).
>
